## Logger

<OptionsTable path="../../packages/logger/src/types.ts" name="Logger" nameLabel="Prop" />

Primary call shape is `message, meta?` for most levels. `error` and `fatal` accept `error, message?`. Broader `(...args: unknown[])` overloads keep the type assignable to cache-style logger contracts; unsupported extra arguments are preserved under `details`, not printf-interpolated.

## Timestamp and format types

| Type / constant | Description |
|-----------------|-------------|
| `TimestampFormat` | `"time"` \| `"time12"` \| `"iso"` \| `"utcString"` |
| `TimestampTimezone` | `"utc"` \| `"local"` |
| `TimestampOption` | `boolean` or `{ enabled: false }` / `{ enabled: true; format?; timezone? }` |
| `ResolvedTimestampConfig` | Resolved `{ enabled, format, timezone }` used by transports |
| `JsonFieldGroup` | `"timestamp"` \| `"level"` \| `"name"` \| `"message"` \| `"meta"` |
| `LevelPrefixKind` | `"symbol"` \| `"badge"` |
| `ActiveLogLevel` | Log levels excluding `"silent"` |
| `DEFAULT_TIMESTAMP_FORMAT` | `"time"` |
| `DEFAULT_TIMESTAMP_CONFIG` | `{ enabled: true, format: "time", timezone: "utc" }` |
| `DEFAULT_FORMAT_OPTIONS` | Full resolved defaults for `formatOptions` |
| `DEFAULT_JSON_FIELD_ORDER` | Default JSON group order |
| `resolveFormatOptions` | Normalizes partial `FormatOptions` into `ResolvedFormatOptions` |
| `mergeFormatOptions` | Deep-merges transport overrides over resolved logger options |
